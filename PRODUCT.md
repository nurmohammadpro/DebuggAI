# Product

## Register

product

## Users

Mixed audience of solo developers and small teams using DeBuggAI for debugging, code generation, and project management. Users are in a work context — focused, task-driven, moving between the dashboard (project overview, settings, browsing work) and the workspace IDE (building and debugging). They value speed, clarity, and minimal friction.

## Product Purpose

DeBuggAI is an AI-powered debugging and code generation tool. The client dashboard serves as the home base: it's where users land to create new projects, browse existing work, manage settings, check credits, and navigate into the full workspace IDE. Success is measured by how quickly users can move from arrival to their actual task.

## Brand Personality

Clean, sharp, capable. The tone is direct and utilitarian — a professional tool that gets out of the way. Confident but not flashy. Think developer tools: Linear, Vercel, Stripe dashboard. Every pixel earns its place.

## Anti-references

- No trendy SaaS clichés: no gradient accents, no glassmorphism, no heavy shadows, no animated illustrations
- No decorative elements that don't serve a function
- No crowded or enterprise-feeling layouts with dense borders and panels
- No "hero-metric" templates (big number, small label, gradient accent)
- Subtle rounded corners only — nothing exaggerated (4–12px range for containers, 6px for buttons/inputs)
- No em dashes

## Design Principles

1. **Function over decoration.** Every element serves a purpose. Nothing is ornamental. If it doesn't help the user make a decision or take an action, remove it.

2. **Generous typography.** Large type is the primary hierarchical signal. Use scale and weight for separation — not borders, backgrounds, or color. Headings are bold and roomy.

3. **Sparse borders.** Whitespace and font weight create structure. Borders are a last resort, used sparingly and thinly.

4. **Flat and precise.** Minimal elevation, subtle rounded corners (4-12px), no shadows. Buttons, icons, and badges are crisp and utilitarian — polished through proportion and alignment, not decoration.

5. **Direct paths.** Reduce steps between the user and their goal. Every click should feel obvious and inevitable. No modal as first thought.

## Accessibility & Inclusion

Aim for WCAG 2.1 AA contrast ratios. The dark theme is the default — ensure text/background pairs meet 4.5:1 minimum. Support reduced motion preferences. No critical information conveyed through color alone.
